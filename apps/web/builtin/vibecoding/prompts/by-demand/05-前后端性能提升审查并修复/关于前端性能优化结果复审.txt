# 前端性能优化结果复审（仅审查版）

【输入】
- scope：复审范围（页面/路由/组件/构建配置）。
- report_path：报告路径（默认 `.conversations/reports/frontend-performance-optimization-review-report.md`）。
- execution_handoff_path：执行步骤交接文件路径（如 `.conversations/pipeline-handoffs/<pipeline_id>/step2-handoff.json`）。
- acceptance_criteria：至少包含“指标达成度、关键交互回归风险、未达标项、复审结论”。
- constraints：审查深度、时间预算、设备/网络档位、是否允许运行分析命令。
- （可留空）handoff_output_dir：默认 `.conversations/pipeline-handoffs/<pipeline_id>/`。

【需要完成的功能】
- 仅做前端性能优化结果复审：核对性能指标是否达成且页面体验无回退（不改代码）。

【注意事项】
- 通用审查基线（必须）：先扫描事实→再分级问题→最后给结论；最多追问 3 个阻塞问题；每条 `acceptance_criteria` 必须有证据；按【输出】原样输出。
- 严格单一职责：仅审查不改代码。
- 必须读取 `execution_handoff_path`，并核对“热点编号 → 改动 → 指标证据”映射完整性。
- 前后对比必须同条件（构建参数/设备档位/网络条件/采样窗口一致），不一致需标记风险。
- 报告必须落盘到 `report_path`；如提供 `handoff_output_dir`，结束时写 `step-summary.md` 与 `step-handoff.json`。

【输出】
- 扫描结论（复审范围、输入来源、指标口径、可信度说明）。
- 问题清单（P0/P1/P2：位置/影响/证据/修复建议/验证方式）。
- 指标达成清单（目标指标 → 实测值 → 达成状态 → 证据）。
- 回归风险清单（首屏/交互/错误态/空态/兼容性）。
- 验证清单（已执行命令 + 结果摘要；未执行项手工步骤）。
- 审查报告（已保存路径 + 报告摘要）。
- 复审结论（通过/需继续优化/阻塞）。
- 流水线交接文件（若提供 `handoff_output_dir`：`step-summary.md` + `step-handoff.json`）。
- 验收映射表（acceptance_criteria → 证据 → 结论）。
- 机器可读摘要（JSON）：{"status":"success|partial|blocked","scope":[],"metric_status":[],"regression_risks":[],"next_actions":[]}。
